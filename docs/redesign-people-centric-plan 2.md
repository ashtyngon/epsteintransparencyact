# Recenter the site on the people ("the Epstein list")

## Context

`epsteintransparencyact` is currently a **news-first** automated aggregator: the homepage leads with a news hero + a 9-article feed, and people are relegated to a 4-item sidebar box. Yet the site already owns a rich **people domain** — 92 sourced, categorized, interlinked profiles; per-person OG images; `Person` JSON-LD; bidirectional article↔person linking — and search demand for this topic centers on *names/the list* (Wikipedia's *List of people named in the Epstein files*, the DOJ letter naming hundreds, the EpsteinExposed DB). Best-in-class comparables (ICIJ Offshore Leaks, EpsteinExposed) are built as **searchable name index → per-person profile → connections/network view**.

The goal: **invert the information architecture so the people ("the list") are the centerpiece**, while keeping all 344 news articles as a supporting section. We are **restructuring and rearranging only — the existing visual design (colors, fonts, components) stays as-is** (per the user's explicit instruction; the earlier "full visual overhaul" idea is dropped).

### Decisions locked with the user
1. **The directory IS the homepage.** The canonical people directory lives at `/` (the searchable/filterable A–Z name index). 
2. **Keep the current look.** No `@theme` palette or font changes. Reuse existing tokens/classes/components everywhere. "Rearrange everything else."
3. **One canonical directory.** Both `/names` and the legacy `/people` index 301-redirect to `/`. Person profiles stay at `/people/{slug}/` (unchanged URLs).
4. **All depth features now:** timeline events on profiles, "frequently appears with" connections, a people-to-people network view, and an auto-grow pipeline that scaffolds profiles for newly-named people.

### Two near-duplicate index pages exist today (the consolidation target)
- `src/pages/names/index.astro` (~440 lines) — the **rich** directory: H1 "Full List of Names in the Epstein Files", A–Z last-name jump nav, category `<select>` filter, A↔Z sort, List/Profiles toggle, `<noscript>` SSR fallback table, `ItemList` JSON-LD of all 92. **This is the basis for the new homepage.**
- `src/pages/people/index.astro` — a simpler category-grouped `PersonCard` grid. Redundant.
- Both link to `/people/{slug}` profiles. Nav points to `/names`; footer points to `/people`.

## Workstream E — Auto-grow the people list (pipeline)

**E.1 `scripts/scaffold-people.ts` (new).** Runs **after fact-check, before cross-link** (so only surviving articles contribute, and new profiles exist before cross-link links them).
- **Candidates:** re-scan surviving article frontmatter (`people[]` raw names + `tags[]`); overlay article-level `confidence`/`newsworthiness` from `scripts/config/relevant.json` (still present pre-cleanup). Re-scanning files (not trusting `relevant.json` alone) automatically respects all deletions.
- **Normalize/dedupe:** slugify; dedupe against existing profiles by **slug AND name AND aliases** (reuse `loadPeople`, 0.4); within-batch dedupe; slug-collision → append tag/role disambiguator (repo precedent: `david-stern-advisor.md`). Never overwrite; `existsSync` guard → idempotent.
- **Thresholds (avoid noise):** scaffold only if appears in **≥2 distinct surviving articles** OR in **1 article with `confidence≥0.85` AND `newsworthiness≥7`**; passes a **personal-name regex** (≥2 capitalized tokens, no digits, 2–4 tokens) + a **non-person stoplist** (Congress, Senate, DOJ, FBI, news orgs, `Inc/LLC/University/Foundation`, etc.) + `UNIVERSAL_NAMES`; **per-run cap = 8** (overflow logged for next run).
- **Generation = thin, review-flagged stub (NOT an AI bio — defamation safety):** `status:'review'`, `autogenerated:true`, `confidence` set; neutral `role` ("Named in Epstein-related reporting"); **`category` default `named-in-documents`** (asserts only "was named") via a conservative tag map (never auto-assign `associate`); templated neutral `shortBio`; `sources` built from the mentioning articles (guarantees ≥1 valid URL → renders as a graph node); `firstMentionedDate` = earliest mention; `notableConnections = ['jeffrey-epstein']` + existing co-mentioned slugs (cap 3, existing-only); `sensitive` defaults **true** when victim/uncertain. Body carries a **review banner** + the **inclusion-disclaimer** text + a "Coverage" list linking the source articles.

**E.2 Wire `.github/workflows/fetch-articles.yml`** — insert a step after "Fact-check", before "Cross-link", running `npx tsx scripts/scaffold-people.ts`. **Add `src/content/people/` to the `git add`** (currently omitted — the single most important non-obvious change; without it, stubs are created in CI and discarded every run).

**E.3 Interactions/guardrails:** auto-profiles become graph nodes for free (cross-link adds their slug to articles the same run → they appear in co-mentions/network). `notableConnections` only ever holds existing slugs (no dangling edges). Note cross-link's `MAX_PEOPLE_PER_ARTICLE=10` may truncate late-appearing new names from a given article (acceptable; raising the cap is a separate easy change). `victim-survivor` resolutions stay `status:'review'` + `sensitive:true`.
